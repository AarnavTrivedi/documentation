# Users

Users are the authors of cards and owners of collections. On this page, we’ll dive into the different verification endpoints you can use to manage Users programmatically. We'll look at how to create verifications. {{ className: 'lead' }}

## The user model

The user model contains all the information about each user, including their email, username, website, and more. 


### Properties


<Properties>
  <Property name="id" type="string">
    The id of the user.
  </Property>
  <Property name="email" type="string">
    The email of the user.
  </Property>
  <Property name="username" type="string | null">
    The username of the user.
  </Property>
  <Property name="website" type="string | null">
    The website of the user.
  </Property>
  <Property name="visible_email" type="boolean">
    Whether the email of the user is visible.
  </Property>
</Properties>


## Update a user {{ tag: 'PUT', label: '/api/user' }}

<Row>
  <Col>

    This endpoint allows you to update yourself. (Must be authenticated to use)

  ### Optional attributes

  <Properties>
    <Property name="username" type="string">
      The username of the user.
    </Property>
    <Property name="website" type="string">
      The website of the user.
    </Property>
    <Property name="visible_email" type="boolean">
      Whether the email of the user is visible.
    </Property>
  </Properties>


  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/api/user">

    ```bash {{ title: 'cURL' }}
      curl -L 'http://api.arguflow.com/api/user' \
        -H 'Content-Type: application/json' \
        -H 'Cookie: ai-editor=your_cookie_value_here' \
        -d '{
            "username": "denssumesh",
        }'
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
    {
      "id": "0c6cc3dc-2069-4f5e-a306-eed3f13ac16e",
      "email": "foobar@gmail.com",
      "username": "denssumesh",
      "website": null,
      "visible_email": false
    }
    ```

  </Col>
</Row>
