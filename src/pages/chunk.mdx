# Chunks

Chunks are the fundamental unit of information in Vault. They represent a single piece of information, such as an article, with an author, file, and formatted chunk content. On this page, we will dive into the different chunk routes you can use to manage and search through chunks programmatically. {{ className: 'lead' }}

## The chunk model

The attachment model contains all the information about a chunk, including the author, file, date created, and formatted chunk content.

### Properties

<Properties>
  <Property name="id" type="string">
    Unique identifier for the chunk.
  </Property>
  <Property name="author" type={<a href="user#the-user-model">User</a>}>
    User object containing the author's id and email.
  </Property>
  <Property name="content" type="string">
    Chunk content in plain text without html formatting.
  </Property>
  <Property name="chunk_html" type="string">
    Chunk content with HTML formatting.
  </Property>
  <Property name="link" type="string">
    The link to the article the chunk is from.
  </Property>
  <Property name="qdrant_point_id" type="string">
    The id of the chunk in Qdrant.
  </Property>
  <Property name="total_upvotes" type="number">
    The total number of upvotes the chunk has.
  </Property>
  <Property name="total_downvotes" type="number">
    The total number of downvotes the chunk has.
  </Property>
  <Property name="vote_by_current_user" type="null | boolean">
    The total number of downvotes the chunk has.
  </Property>
  <Property name="created_at" type="timestamp">
    The date and time the chunk was created.
  </Property>
  <Property name="updated_at" type="timestamp">
    The date and time the chunk was last updated.
  </Property>
  <Property name="tag_set" type="null | string">
    Tags associated with the chunk.
  </Property>
  <Property name="file_id" type="null | string">
    The id of the file the chunk was uploaded from.
  </Property>
  <Property name="file_name" type="null | string">
    The name of the file the chunk was uploaded from.
  </Property>
  <Property name="private" type="boolean">
    Whether the chunk is visible to other users.
  </Property>
  <Property name="verification_score" type="null | number">
    The similarity score of the chunk to the webpage it is linked to.
  </Property>
</Properties>

## Create a Chunk {{ tag: 'POST', label: '/chunk' }}

<Row>
  <Col>

    This route allows you to create a new chunk. You must be authenticated to use it successfully.

    ### Request body

    The request body should be a JSON object with the following fields, all of which are optional.

    <Properties>
      <Property name="chunk_html" type="string">
        The chunk content with html formatting. If provided it must contain more than 70 words.
      </Property>
      <Property name="link" type="string">
        The link to the article the chunk is from.
      </Property>
      <Property name="tag_set" type="array">
        The tags you wish to associate with the chunk.
      </Property>
      <Property name="private" type="boolean">
        Whether the chunk is visible to other users.
      </Property>
      <Property name="file_uuid" type="string">
        The id of the file the chunk was uploaded from.
      </Property>
      <Property name="metadata" type="object">
        Additional attributes to add to the chunk.
      </Property>
    </Properties>

    ### Response

    If the response is successful it will return a 200 status code with the created chunk and whether it is a duplicate or not.

    <Properties>
      <Property name="chunk_metadata" type={<a href="#the-chunk-model">Chunk</a>}>
        The chunk object that was created.
      </Property>
      <Property name="duplicate" type="boolean">
        Whether the chunk is a duplicate of another chunk.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/api/chunk">

    ```bash {{ title: 'cURL' }}
            "metadata": {
              "email": "createchunk@example.com",

            }
          }'
      ```

## Update a chunk {{ tag: 'PUT', label: '/chunk/update' }}

<Row>
  <Col>

    This route allows you to update a chunk from the database using its id.

    ### Required attributes
    <Properties>
      <Property name="chunk_uuid" type="string">
        The chunk id of the chunk you wish to update.
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="link" type="string">
        The new link for the chunk.
      </Property>
      <Property name="chunk_html" type="string">
        The new chunk_html for the chunk.
      </Property>
      <Property name="private" type="boolean">
        The new link for the attachment.
      </Property>
    </Properties>

      ### Response

    If the response is successful it will return a 204 status code with no body.

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="PUT" label="/v1/update">

    ```bash {{ title: 'cURL' }}
      curl --location --request PUT 'https://api.trieve.ai/api/chunk/update' \
          --header 'Content-Type: application/json' \
          --header 'Cookie: ai-editor=your_cookie_value_here' \
          --data '{
              "chunk_uuid": "0b029764-6083-44ec-a741-7a278e7b1459",
              "private": false
          }'
    ```

    </CodeGroup>

  </Col>

## Semantically search for a chunk {{ tag: 'POST', label: '/api/chunk/search/:page' }}

<Row>
  <Col>

    This route allows you to search the database by converting your query into a vector and searching for the most similar vectors in the database, returning the top 25 chunks. You go through all the pages of chunks by incrementing the page number in the request URL.

    ### Required attributes
    <Properties>
      <Property name="content" type="string">
        The query you wish to search for.
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="tag_set" type="array">
        Tags to filter the search by.
      </Property>
      <Property name="link" type="array">
        Which links to include in the result.
      </Property>
    </Properties>

    ### Response

    If the response is successful it will return a 200 status code with an 2D array of chunk objects and the collisions and the total pages of chunks.

    <Properties>
      <Property name="score_chunks" type={<text>[<a href="chunk#the-chunk-model">Chunk</a>]</text>}>
          A 2D array of chunk objects and their collisions.
      </Property>
      <Property name="total_pages" type="integer">
          The total number of pages of chunks.
      </Property>
    </Properties>

  </Col>

## Get count of chunks currently in the db {{ tag: 'GET', label: '/api/chunk/count' }}

<Row>

    ### Response

    If the response is successful, it will return a 200 status code with the chunk_count.

    <Properties>
      <Property name="total_count" type="number">

      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="GET" label="/api/chunk/count">

    ```bash {{ title: 'cURL' }}
      curl --G 'https://api.trieve.ai/api/chunk/count' \
        --header 'Content-Type: application/json'
    ```

    </CodeGroup>
    ```json {{ title: 'Response' }}
      {
        "total_count": total_count
      }
    ```

  </Col>
